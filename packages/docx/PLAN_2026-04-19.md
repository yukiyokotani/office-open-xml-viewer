## タスク (優先度順)

### P1 — 小さくて影響が大きい機能

1. **Superscript / subscript**
   - Parser: `w:vertAlign val="superscript|subscript"` を `RunFmt.vert_align` として追加
   - Types: `TextRun.vertAlign: 'super' | 'sub' | null`
   - Renderer: フォントサイズを ×0.65、y オフセット (super は -0.35em、sub は +0.15em) で描画
   - 見積: 1h

2. **Hyperlinks のレンダリング整備**
   - Parser: 既に `w:hyperlink` 内の run を検出して `isLink` を立てている想定だが確認
   - relationships.xml から URL を引いて `TextRun.hyperlink?: string` に格納
   - Renderer: `isLink` の run は色 `#0563C1` + underline をデフォルトに (run 自体に color / u 指定があればそれを尊重)
   - 見積: 1.5h

3. **Tab stops (明示的 w:tabs)**
   - Parser: `pPr/w:tabs/w:tab` を `ParaFmt.tab_stops: Vec<TabStop>` として追加
     - `pos: pt`, `val: 'left'|'center'|'right'|'decimal'`, `leader: 'none'|'dot'|...`
   - Types: `DocParagraph.tabStops: TabStop[]`
   - Renderer: layoutLines で `\t` 文字を検出、次のタブ位置まで空白を確保する関数に置換
     - 現状 `\t` はナンバリング用途のみなので、汎用化が必要
   - 見積: 2h

### P2 — 中規模

4. **段落スタイル (Heading 1–6 / Normal) の目視検証と修正**
   - StyleMap 自体は実装済み。サンプル docx で Heading 1–6 が期待通り描画されるか確認
   - よくある不備を潰す:
     - `w:keepNext` / `w:keepLines` (今はスキップ)
     - Heading に lang-specific フォントが入ってるケース
     - outline 番号 (Heading に `numPr` が紐付く場合)
   - 見積: 1.5h

5. **Section breaks (複数セクション対応)**
   - 現状: `body` 末尾の `sectPr` のみ読んでいる
   - 改修: 段落内の `pPr/sectPr` も認識し、セクション区切りとして扱う
   - MVP: ページブレーク相当として扱う (ページサイズ/マージン変更は後回し)
   - Types: `BodyElement` に `'sectionBreak'` バリアント追加、または `pageBreak` で代用して section 切替は無視するか判断
   - 見積: 1h

### P3 — 余裕があれば着手

6. **Footnotes / endnotes (MVP)**
   - `word/footnotes.xml` / `word/endnotes.xml` を読み込み
   - 本文中の `w:footnoteReference id="N"` を上付き数字でレンダリング
   - ページ下部に該当する footnote 本文をフッターより上に配置
   - 規模大。スコープを「上付き数字のみ表示、本文は非表示」に限定すれば 2h 程度

## 作業手順

各タスクは以下の順で進める:

1. **Red**: サンプル docx を用意 (Word で手作りするか既存ファイルを流用)。期待描画を参照 PNG として確保し VRT spec に追加
2. **Green**: Parser (Rust) → Types (TS) → Renderer (TS) の順で実装
3. **WASM ビルド**: `cd packages/docx/parser && wasm-pack build --target web --out-dir ../src/wasm`
4. **Storybook で目視**: `pnpm storybook` → DocxViewer / DebugJson で確認
5. **VRT 実行**: `npx playwright test packages/docx/tests/visual`
6. **Commit**: 機能単位で分割コミット。コミットメッセージは英語 (public repo)

## 前提 / 注意点

- サンプル .docx はコミットしない (`public/` は `.gitkeep` のみ)
- 参照 PNG (`tests/visual/references/`) は **ユーザー指示時のみ追加**、自動更新しない
- `packages/core` / `packages/pptx` / `packages/xlsx` は読み取り専用
- コミット前に `pnpm --filter @ooxml/docx typecheck` でビルドエラーをチェック
